SYNTHESIS — JFEcon house style for Paper 3 v1
========================================================================

Length statistics across the 6 JFEcon papers in our sample:
  Pages range:        26 - 74   (median ~48)
  Word count range:   9k - 36k  (median ~17k)
  Tables range:       5 - 32    (median ~13)
  Figures range:      7 - 14    (median ~11)

Bibliography: ALL 6 use author-year (Smith 2020) — JFEcon house style.
Section depth: typically 2 levels (2.1, 2.1.1).

Canonical section template observed across JFEcon papers:
  1. Introduction               (motivation + contribution + roadmap)
  2. Methodology / Theoretical Framework / Models
       2.1, 2.2, ... subsections per model class
  3. Data Description           (named source, sample period, cleaning)
  4. Empirical Results
       4.1 In-sample / one-step-ahead
       4.2 Multi-step / longer-horizon
       4.3 Robustness checks
  5. Conclusions / Concluding Remarks
  References (author-year, alphabetical)
  A. Appendix(es)               (additional tables, proofs, hyperparameter detail)

Hedging language patterns observed (use these in v1):
  - 'may', 'might', 'could'
  - 'tend to outperform' (not 'always outperforms')
  - 'appears to', 'seems to', 'suggests'
  - 'robust', 'robustness checks', 'caveat'
  - 'we cannot reject', 'fail(s) to reject'
  - 'in our sample' / 'in this test period' (avoid universal claims)

Reproducibility statements: JFEcon papers describe data in prose within the
Data section (not a labeled 'Data Availability Statement' like MDPI). For v1
we should follow MDPI/journal-of-financial-data-science conventions and
include both: a clear Data section AND a labeled Code/Data Availability
block — this is where Paper 3 v1 can DIFFERENTIATE from JFEcon norms by
modelling better disclosure practice.

Tables: significance stars (*, **, ***), SE in parentheses, bold values for
winners, Panel A / Panel B / Panel C subdivisions for split samples.

Figures: matplotlib-style line plots, time series, Q-Q plots; minimal use
of color (still common in JFEcon).

DIFFERENTIATION FOR PAPER 3 v1 (do BETTER than JFEcon norms):
  - Add a labeled 'Data and Code Availability' block (MDPI style)
  - Make the package vol-eval the open-source artifact (Paper 3's edge)
  - Include the audit-script convention as a paper-level reproducibility
    feature (not seen in any of the 6 JFEcon papers)
  - Apply DM/MCS/SPA to the 30-paper sample explicitly (Phase 2 finding)
